**1. 核心用户分层**

-   **知识工作者（主力，约占45%）**
    典型角色：程序员、产品经理、科研/教育从业者、媒体人。
    高频场景：代码调试、技术方案设计、文档润色、论文辅助、热点深挖。

-   **高净值学习者（约占25%）**
    典型角色：备考学生、终身学习者、高管。
    高频场景：跨学科知识串联、外文精读、复杂逻辑推演。他们看重上下文长度（1M超长记忆）和联网搜索的权威性。

-   **效能极客与工具整合者（约占15%）**
    典型角色：独立开发者、自动化爱好者、中小企业主。
    高频场景：API调用、将DeepSeek集成进工作流（如客服、数据清洗）。他们是对价格最敏感的一群，受“免费且开源”吸引而来。

-   **价格敏感与尝鲜大众（约占15%）**
    在此解决一次性复杂需求（如法律合同草拟、旅行规划），或因社交媒体热点而来的体验者。他们容易沉淀为“沉默用户”，只在真有难题时才回来。

**2. 行为与心理特征**

-   **理性实用主义**：目的性极强，不爱闲聊，不追求人格化陪伴。输入提示词通常很长、结构化，视其为“思维外挂”。
-   **“白嫖”技术红利**：不仅在财务上（免费），更在智力上。他们清楚顶级模型的能力，来DeepSeek是为了打破信息差。
-   **无情感负担的工具关系**：用户很少尝试“越狱”或建立拟人关系，产品气质鼓励纯粹的功能性使用。

**3. 典型使用场景画像**

-   **场景一：凌晨代码审计**。用户粘贴上千行报错日志，DeepSeek秒级定位并给出修正代码，用户感叹“省下买Copilot的钱”。

**与竞品的差异定位**：ChatGPT用户偏“全能创作”，Kimi用户强于“长文档消费”，而DeepSeek用户则是**深度推理和代码等高难度任务的“硬核挖掘者”**。

现有方案（浏览器插件/通用脚本）普遍存在**数据抓取不稳定、Markdown格式丢失“深度思考”内容、缺乏批量交互与排重能力**等问题。本项目旨在提供一个从数据提取到知识沉淀的完整、专业的“提取层”解决方案。

**核心优势**

-   **🚀 稳定可靠的API直连**：模拟官方客户端行为，直接从浏览器Token获取数据。**不依赖网页DOM**，无失效风险，是当前最稳定的获取方案。
-   **💡 终端交互的“降维体验”**：基于Bubbletea构建全屏TUI，支持**键盘全操控、分页按需加载、精准勾选**。兼具脚本的效率和应用的体验，让技术用户可以优雅地管理海量对话。
-   **📝 无损格式还原**：深度优化Markdown模板，**完美还原DeepSeek独有的“深度思考”过程**和对话结构。专为Obsidian等笔记软件优化，便于直接构建个人AI知识库。
-   **⚡ 智能本地排重**：内置排重引擎，在导出前自动比对本地文件，**已备份的对话自动跳过**，显著节省时间和API请求。
-   **💻 原生跨平台单文件**：Go语言构建，单二进制文件，无任何运行时依赖，覆盖Windows, macOS, Linux (amd64/arm64)。

**快速开始**

```bash
# 下载对应平台的二进制文件
# 1. 从浏览器获取你的DeepSeek Token
# 2. 运行程序，在TUI界面中输入Token并开始浏览导出
./deepseek-exporter
```

在DeepSeek网页端聊了几十轮，代码方案超精妙、论文思路瞬间打通、AI的“深度思考”过程满是宝藏。但聊完，它们就静静躺在左侧列表里吃灰。想导出整理，官方只给个压缩包，解压是看不懂的JSON，手动复制到手抽筋……

一个**终端界面(TUI)** 工具。别被“终端”吓到，你可以把它理解成一个**专门为DeepSeek设计的、纯键盘操作的超级备份管理器**。没有花里胡哨的界面，但效率极高，功能专精。

**💡 它凭什么比插件和脚本好用？**

-   **稳过“宫斗”插件**：市面上的浏览器插件，靠抓取网页元素运行，DeepSeek页面一更新就报废。我直接通过官方API接口获取数据，从根源上保证稳定。你只需要首次从浏览器拿一下Token，后面的故事就交给它。
